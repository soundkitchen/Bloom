# 2026-06-09 — 保存・読み込み・PNG 書き出し(M1 完了)

M1 最後のピース。feature ブランチ `feature/save-load`。

## 設計判断: ラスタ形式の `.bloom`

ドキュメントは**エンジンの実状態(各レイヤーの `D` + メタデータ)をそのまま保存するラスタ形式**にした。ストローク履歴を持って再生する案もあったが、連続的な流体シミュレーションでは再生が脆い(undo と同じ理由)。エンジンの状態 = レイヤーの deposit + visible/opacity/name + active/counter なので、それを素直に直列化する。

バイナリ書式(リトルエンディアン): magic `"BLM1"` + version + width/height + layerCount/active/counter + 各層 { name, visible, opacity, deposit raw }。エンディアンはバイト単位で明示(移植性)。

- **ウェット(W/P)は保存しない** = 乾いた絵を保存する。紙テクスチャは寸法から決定的なので保存不要
- 読み込みは寸法一致を検証(グリッドは生成時固定)。不一致・不正 magic は `EngineError.documentFormat` を投げ、アプリ側で NSAlert 表示
- 読み込み時に undo/redo 履歴とウェットをリセット

## アプリ

ファイルメニュー: 開く(Cmd+O)/ 保存(Cmd+S)/ 別名で保存(Cmd+Shift+S)/ PNG を書き出す(Cmd+E)。NSOpenPanel/NSSavePanel。現在のドキュメント URL を保持し、ウィンドウタイトルに反映。
