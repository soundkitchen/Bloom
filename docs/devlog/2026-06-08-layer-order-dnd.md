# 2026-06-08 — レイヤーの順序合成 + D&D 並べ替え

「レイヤーを D&D で入れ替えたい」という要望から。ただし入れ替えが意味を持つには合成の作り替えが要るので、2 ステップで実装した。

## 前提: そのままでは並び替えても絵が変わらない

それまでのレイヤー合成は**吸光度の加算**(`paper·exp(-Σ各層)`)で、加算は順序非依存。つまり並び替えても見た目が一切変わらなかった。これは半透明グレーズとしては物理的に正しいが、Photoshop 的な「上が下を覆う」とは違う。→ まず順序が効く合成を入れることにした。

## ステップ1: アフィングレーズ合成

最初に素直なプリマルチプライド over を試したが、**全体が washed-out になった**。原因: over は「被覆(alpha)」と「色味(tint)」を 1 つの alpha に押し込むが、水彩ウォッシュは被覆が低くても色味は強い。低 alpha だとほとんど色が乗らない。

そこで **アフィングレーズモデル**に切り替え。各層は下の色 `r` を `r → a·r + b` に変換する:

- `T = exp(-D)`(透過色)、`occ = opacity·(1 - exp(-k·輝度))`(被覆)
- `a = T·(1-occ)`, `b = T·occ`
- 薄い顔料(`occ≈0`)→ `r→T·r` の純粋な乗算フィルタ = **全力で色を付けつつ下が透ける**(washed-out 解消)
- 濃い顔料(`occ≈1`)→ `r→T` で下を置き換える(**順序が効く**)

アフィン変換は合成できるので、下/上の可視層を `(A,B)` 各 1 組へ畳み込める(`belowA/B`, `aboveA/B`)。畳み込みは GPU の `compositeLayerKernel` を下→上に逐次 dispatch(`memoryBarrier` で順序保証)、レイヤー操作時のみ再構築。

検証(`--demo-layers`、濃いめのストローク):

| 赤が上(both) | 青を最前面へ(reordered) |
|---|---|

中央の X 交差で上下が入れ替わる = 順序が効いている。薄いウォッシュだと差は微小(= 正しい挙動)。

## ステップ2: NSTableView で D&D + 不透明スライダ

インスペクタの自作スタックリストを **NSTableView** に載せ替え:

- 行の **D&D 並べ替え**(`pasteboardWriterForRow` / `validateDrop`(`.above` に矯正)/ `acceptDrop`)→ `moveLayer(fromRow:toRow:)`
- 選択 → アクティブ化、目アイコン → 表示切替、＋/🗑、**選択層の不透明スライダ**
- プログラム選択 → `selectionDidChange` の無限ループは `isReflecting` フラグで抑止
- `moveLayer` はアクティブ層を **deposit バッファの同一性**で追従させる(`===`)
