# 2026-06-10 — MP4 動画書き出し(M2 積み残し)

M2 の積み残しだった「動画(MP4)書き出し」を実装した。feature ブランチ `feature/mp4-export`。
これで書き出しは GIF / MP4 / スプライトシート / PNG 連番の 4 形式が揃い、M2 の `⬜ 動画(MP4)` が片付いた。

## それまでの状態

アニメーション書き出しは `BloomCore/AnimationExport.swift`(`SimulationEngine` の extension)に GIF / スプライト / 連番の 3 つがあった。いずれも共通ヘルパ `renderAllFrames()`(全フレームへ `goToFrame` しながら `renderFrameCGImage()` で CGImage 化。オニオン無効化・currentFrame 復元は `defer`)で全フレームの `CGImage` を得て ImageIO/CoreGraphics で書く、という形。MP4 も「全フレーム CGImage → エンコーダに流す」で同型に収まる。

## やったこと

### 1. コア `exportMP4(to:fps:)`

`AVAssetWriter`(`.mp4`)+ `AVAssetWriterInput`(H.264)+ `AVAssetWriterInputPixelBufferAdaptor`。`renderAllFrames()` の各 `CGImage` を `CVPixelBuffer`(BGRA)へ描き込んで append する。書き出し順は `startWriting` → `startSession(.zero)` → append(PTS = `CMTime(value: i, timescale: fps)`)→ `markAsFinished` → `finishWriting`。

ピクセルフォーマットは `kCVPixelFormatType_32BGRA`。これは `renderFrameCGImage` の CGContext(`premultipliedFirst | byteOrder32Little`)および元の Metal テクスチャ `.bgra8Unorm` と一致するので、色順の変換は不要。`CGContext` の `bytesPerRow` はプールの実測値(`CVPixelBufferGetBytesPerRow`)を使う(アラインで `width*4` より大きいことがある)。

### 2. UI / メニュー

ファイルメニューに「MP4 を書き出す…」(**Cmd+Shift+M**)を追加。素の Cmd+M は macOS の「しまう(Minimize)」と衝突するので避けた。ハンドラは `exportGIF` を雛形に `NSSavePanel`(`.mpeg4Movie`)→ `engine.exportMP4(to:fps:)`。

fps は **タイムラインの再生速度(`playFps`)に追従**させた。再生で見た速さのまま動画になるのが直感的だから。GIF は従来どおり 12fps 固定(将来 `playFps` 化してもよい)。

### 3. 検証モード

`--demo-anim` の書き出しブロックに `anim.mp4` を 1 行足した。新規デモモードは作っていない。

## 判断・ハマりどころ

- **偶数寸法(H.264 要件)**: H.264 は奇数の幅/高さで失敗しうる。`gridWidth/Height` は `max(width,8)` で偶数保証が無いため、出力幅高を `& ~1` で**偶数へ切り捨て**(右端/下端 1px を落とす)。`AVVideoWidthKey/HeightKey`・バッファ寸法・`CGContext` の draw 先をすべて偶数で一貫させる。1px 切り捨ては動画では実用上問題にならないと判断(GIF/スプライトは従来どおり全画素を出す)。
- **同期 API の維持**: 既存 export は同期 `throws`。非同期の `finishWriting(completionHandler:)` を `DispatchSemaphore` で待つ。完了ハンドラは AVFoundation 内部キューで発火するため、`@MainActor`(メインスレッド)で `wait()` してもデッドロックしない。GIF の ImageIO 同期ブロックと同水準で、書き出し中に UI が一瞬固まるのみ。
- **`isReadyForMoreMediaData`**: オフライン(`expectsMediaDataInRealTime = false`)では基本すぐ ready。少数フレーム前提で `Thread.sleep(0.002)` 付きのスピン待ちにした。将来フレーム数が増えるなら `requestMediaDataWhenReady` の pull 型へ。
- **リンク**: `import AVFoundation` だけで Swift の auto-link が効き、`project.yml` は無改変でビルドが通った(CoreMedia/CoreVideo は再エクスポート)。
- **既存ファイル**: `AVAssetWriter(outputURL:)` は出力先が存在すると失敗するので、init 前に `try? removeItem(at:)`。
